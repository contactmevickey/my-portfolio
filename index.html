About

Nine years, one thread: systems that outlast their first version

I've spent my career across the full lifecycle of backend platforms — from rebuilding a C++ middleware system in Java at Verizon, to leading microservices and API contract design at Freshworks. What ties it together is a focus on systems that other engineers can build on without re-learning them: clear contracts, sane defaults, and architecture decisions someone can still explain two years later.

As a Technical Lead, I split my time between system design, technical roadmaps, and mentoring the engineers who turn those decisions into shipped product. I've recently folded AI-assisted development — Cursor and Claude — into that workflow, using prompt engineering for code generation, test scaffolding, and documentation, to move faster without trading away engineering quality.
